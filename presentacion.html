Apache Spark

El Motor Definitivo de Procesamiento de Datos a Gran Escala

📦 ¿Qué es Apache Spark?

🚀 Procesamiento Distribuido

Ejecuta tareas de computación pesadas dividiendo el trabajo entre múltiples máquinas (clústeres) de forma simultánea.

⚡ Velocidad "In-Memory"

A diferencia de Hadoop MapReduce (que escribe en disco), Spark usa la memoria RAM, siendo hasta 100x más rápido.

🧩 Framework Unificado

Une Batch Processing, Streaming en tiempo real, Machine Learning (MLlib) y consultas SQL en una sola API.

🐍 ¿Cómo usarlo? (PySpark)

PySpark es la API de Python. Todo comienza inicializando la SparkSession:

from pyspark.sql import SparkSession

# Crear la conexión (Session) con Spark
spark = SparkSession.builder \
    .appName("MiPrimeraApp") \
    .master("local[*]") \
    .getOrCreate()

# ¡Listo! Ahora puedes usar 'spark' para leer y procesar datos.

🔌 Formas de Acceder a los Datos

1. Archivos Locales (CSV, Parquet)

df = spark.read.csv("datos/clientes.csv",
    header=True, inferSchema=True)

2. Bases de Datos (JDBC)

df = spark.read.format("jdbc") \
    .option("url", "jdbc:postgresql://...") \
    .option("dbtable", "usuarios") \
    .load()

3. MinIO (S3) usando Boto3 🌟

MinIO es S3 "on-premise". Si requerimos usar boto3, leemos el stream directo a memoria RAM sin tocar el disco:

import boto3
from io import BytesIO

# 1. Conectar Boto3 a MinIO
s3_client = boto3.client('s3',
    endpoint_url='http://localhost:9000',
    aws_access_key_id='minioadmin',
    aws_secret_access_key='minioadmin')

# 2. Descargar a memoria (BytesIO)
resp = s3_client.get_object(
    Bucket='mi-bucket', Key='datos.parquet')
stream = BytesIO(resp['Body'].read())

# 3. Cargar stream en PySpark
df_minio = spark.read.parquet(stream)

⚙️ Tipos de Cálculos en Spark

Spark funciona con Evaluación Perezosa (Lazy Evaluation). Todo el proceso se divide en dos fases:

🔄 TRANSFORMACIONES

Preparan el cálculo, pero NO se ejecutan hasta que se llama a una Acción.

  • filter(): Filtrar filas.
  • groupBy().agg(): Agrupar y calcular métricas.
  • join(): Cruce de tablas masivas.
  • withColumn(): Crear columnas nuevas.

🎯 ACCIONES

Disparan TODO el grafo de transformaciones y devuelven un resultado.

  • show(): Muestra una muestra en pantalla.
  • count(): Cuenta el total de filas.
  • write(): Guarda el resultado final en disco, MinIO o BD.

🧠 Cálculos Avanzados Integrados

  • Spark SQL: Permite hacer consultas SQL puras: spark.sql("SELECT ...") sobre DataFrames.
  • Machine Learning (MLlib): Entrenar modelos (Regresión, Random Forest) distribuidos en clústeres.
  • GraphFrames: Cálculos de grafos (rutas más cortas, detección de comunidades en redes sociales).

🛡️ Tolerancia a Fallos (Resiliencia)

Si un servidor se apulta mientras procesa 1 TB de datos, Spark no pierde el trabajo. Gracias a los RDDs (Conjuntos de Datos Distribuidos Resilientes), Spark recuerda el origen de los datos y recalcula únicamente la pieza que falló en otro servidor sano.
